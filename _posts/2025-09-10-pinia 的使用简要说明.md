## 语法（组合式）

——便于理解，可能会和选项式语法进行等价。

### defineStore

```jsx
export const <接口函数名> = defineStore('<唯一标识id>', () => { <详细定义> })
// 接口函数：在组件中调用它，就能拿到这个 store 的实例
// 唯一标识id：用来连接 store 和 devtools，在 devtools 或其他地方识别
// 注意 <详细定义> 处要用箭头函数

// 举例：
export const useAlertsStore = defineStore('alerts', {
  // 其他配置...
})
```

### 详细定义-参数

1. **响应式属性**（对应 `state`）：

```jsx
const <属性名> = ref(<属性值>)

// 举例：
const count = ref(0)
const name = ref('Eduardo')
// 等价于 state: () => ({ count: 0, name: 'Eduardo' })
```

1. **计算属性**（对应 `getters`）：

```jsx
// computed()
// 举例：
const doubleCount = computed(() => count.value * 2)
```

1. **函数方法**（对应 `actions`）

```jsx
// function()
// 举例：
function increment() {
    count.value++
  }
```

1. **return**

> 要让 pinia 正确识别 `state`，你**必须**在 setup store 中返回 **`state` 的所有属性**。
>

```jsx
...
```

## 功能分类

- `state`：核心数据，store 对象的属性值。
- `getters`：只读取&处理（类似 Vue 的 computed）。
- `actions` ：修改 state 的方法；同时可进行异步操作、业务逻辑处理 .etc。
